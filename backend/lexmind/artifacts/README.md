# LexMind Artifact Management System

The Artifact Management System tracks every output generated by the processing pipeline. Artifacts are immutable products of processing stages -- OCR output, parsed text, embeddings, graphs, timelines, reports, and exports.

No producers. No storage implementation. Only the artifact framework.

---

## Artifact Types

| Type | Description |
|---|---|
| `ORIGINAL_DOCUMENT` | Source file (PDF, DOCX, image) |
| `OCR_TEXT` | Extracted text from OCR |
| `OCR_LAYOUT` | Layout analysis output |
| `PARSED_DOCUMENT` | Structured document parse |
| `ENTITIES` | Extracted named entities |
| `RELATIONSHIPS` | Entity relationships |
| `CHUNKS` | Text chunks for embedding |
| `EMBEDDINGS` | Vector representations |
| `VECTOR_INDEX` | FAISS/similar index |
| `KNOWLEDGE_GRAPH` | Knowledge graph structure |
| `TIMELINE` | Temporal event extraction |
| `SUMMARY` | Document summary |
| `REPORT` | Generated report |
| `EXPORT` | User-facing export |
| `CACHE` | Intermediate cache |
| `THUMBNAIL` | Image thumbnail |
| `PREVIEW` | Document preview |
| `LOG` | Processing log |
| `METRICS` | Performance metrics |

## Lifecycle States

```
REGISTERED -> AVAILABLE -> SUPERSEDED -> ARCHIVED -> DELETED
    |                                              ^
    v                                              |
  INVALID ---------------------------------------->+
```

Terminal state: `DELETED`.

---

## Versioning

Artifacts are immutable. Any modification creates a new version:

```
artifact v1 (checksum: abc123) -> superseded
artifact v2 (checksum: def456) -> available
```

Old versions remain accessible via `version_history()`.

---

## Dependency Graph

```
original.pdf -> ocr.json -> parsed.json -> entities.json
                                         -> chunks.parquet -> embeddings.f16 -> faiss.index
```

- Parent: upstream artifact
- Child: downstream artifact
- Cycle detection built-in
- Topological ordering available

---

## Lineage

Full provenance chain is tracked:

```
original.pdf -> ocr.json -> parsed.json -> entities.json -> chunks.parquet -> embeddings.f16 -> faiss.index
```

Every step is traceable via `LineageTracker.ancestors()` and `descendants()`.

---

## Architecture

| Component | Description |
|---|---|
| `Artifact` | Aggregate root with lifecycle + versioning |
| `ArtifactManager` | Orchestrates create/validate/supersede/archive/delete |
| `ArtifactRegistry` | Protocol for persistence |
| `DependencyGraph` | In-memory DAG of artifact relationships |
| `LineageTracker` | Provenance chain traversal |
| `ArtifactManifest` | Serializable manifest value object |
| `ArtifactMetadata` | Immutable metadata value object |

---

## Storage Independence

The artifact framework is completely storage-agnostic:
- `storage_uri` is a string reference (e.g. `s3://bucket/key`)
- No I/O in any interface or model
- Registry implementations handle actual persistence
