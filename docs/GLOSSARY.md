# LexMind Glossary

## Core Concepts

**LexMind** — AI-powered legal research platform for document analysis, knowledge extraction, and case management.

**Chunk** — A segment of a document created during the chunking process, optimized for embedding and retrieval.

**Embedding** — A numerical vector representation of text, capturing semantic meaning for similarity search.

**Hybrid Search** — Combined retrieval using both keyword (BM25) and semantic (vector) search.

**Knowledge Graph** — A graph structure representing entities and their relationships extracted from documents.

**Entity** — A named element extracted from documents (person, organization, date, legal concept, etc.).

**Timeline** — Chronological ordering of events extracted from legal documents.

**Evidence** — A document or excerpt relevant to a legal question or claim.

**Vector Store** — Database optimized for storing and querying high-dimensional vectors.

**Retriever** — Component that finds relevant documents for a given query.

**Reranker** — Model that re-scores retrieved results for improved relevance ranking.

**Context Builder** — Component that assembles retrieved chunks into an LLM-ready context window.

**Prompt Builder** — Component that constructs prompts for LLM inference.

**LLM Provider** — Abstraction layer for language model APIs (OpenAI, Anthropic, local models).

**OCR Provider** — Abstraction layer for optical character recognition engines.

**Plugin** — Modular extension that adds functionality to LexMind.

**Workspace** — A project container organizing documents, indices, and configurations.
