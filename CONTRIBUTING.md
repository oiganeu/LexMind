# Contributing

## Philosophy
Open-source first.

## Branch Strategy
- `main`
- `develop`
- `feature/*`
- `bugfix/*`
- `release/*`
- `hotfix/*`

## Commit Convention
Conventional Commits: `feat:`, `fix:`, `docs:`, `refactor:`, `perf:`, `build:`, `ci:`, `test:`, `chore:`.

## Review Process
Squash merge policy.
